## ADR-0001: Multitenancy via Catalog DB + Project DB

### Контекст
Нужно обслуживать несколько независимых проектов с разными пользователями, правами и объемами данных, при этом не смешивать данные и упростить разграничение доступа.

### Решение
- Ввести **Catalog DB** для глобальной конфигурации (UID/проекты/схемы/правила).
- Для каждого проекта — отдельную **Project DB** (`scada_<slug>`).
- Выбор Project DB делается по `projectSlug` (HTTP) или по UID→projects (TCP).

### Последствия
Плюсы:
- естественная изоляция данных и прав,
- проще выполнять “разные политики хранения” на уровне БД проекта,
- масштабирование по проектам.

Минусы:
- сложнее миграции (много БД),
- нужен менеджер подключений и контроль “утечек” коннектов,
- требуется дисциплина управления DSN.

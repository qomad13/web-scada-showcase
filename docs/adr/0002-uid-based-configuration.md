## ADR-0002: UID-based Global Configuration

### Контекст
Есть станции одного “типа” с одинаковой структурой пакетов, набором тегов, правилами аварий и карточками UI. Нужно переиспользование конфигурации без дублирования на каждый проект.

### Решение
- Ввести сущность **UID** как “профиль типа станции”.
- Станция отправляет UUID → система определяет UID и подтягивает конфигурацию из каталога.
- UID может быть связан с несколькими проектами (routing в TCP ingest).

### Последствия
Плюсы:
- меньше дублирования конфигурации,
- быстрее подключать новый проект/станцию,
- конфигурацию можно менять централизованно.

Минусы:
- нужно версионирование схем/правил (контроль совместимости),
- критично качество валидации данных и “безопасные дефолты”.

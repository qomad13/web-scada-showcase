## Документация (Showcase)

Краткие опорные материалы, которые можно безопасно публиковать. Для деталей — ссылаться на существующую `docs/` исходного проекта, но без копирования чувствительных фрагментов.

### Что включить сюда
- Обзор архитектуры (high-level) и поток данных.
- Роли и границы ответственности (support, superadmin, admin, user).
- UID-based подход и мультитенантность (каталог + проектные БД).
- Поток: прием пакета → декодирование → обработка аварий → запись → realtime push.
- Диаграммы: архитектура, data flow, компоненты (PNG/SVG в `docs/assets/`).

### Что не включать
- Реальные топологии, IP, домены, CIDR.
- Реальные UUID/названия клиентов, реальные payload-примеры.
- Реальные параметры алертов/лимитов/ретраев.

### Структура (пример)
- `01-architecture.md` — high-level overview и диаграммы.
- `02-data-flow.md` — путь пакета и жизненный цикл события.
- `03-multitenancy.md` — каталог/проектные БД, projectSlug routing.
- `04-security.md` — аутентификация (JWT), валидация DTO, CORS, rate/size limits.
- `05-reliability.md` — ретраи, идемпотентность, watchdog/health, graceful shutdown.
- `DIAGRAMS.md` — диаграммы (Mermaid).
- `KEY_DECISIONS.md` — ключевые решения/компромиссы.
- `DEMO_WALKTHROUGH.md` — короткий walkthrough (сквозной сценарий).
- `REDACTION.md` — политика санитизации.
- `adr/` — короткие ADR (почему так).
- `assets/` — диаграммы (без чувствительных деталей).

### Примечание
Для демонстрации кода в текстах использовать синтетические примеры (demo-UID, demo-project).

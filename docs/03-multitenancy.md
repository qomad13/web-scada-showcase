## Мультитенантность и UID-based подход

Модель:
- Catalog DB (`scada_catalog`) хранит UID, теги, правила, проекты, DSN.
- Project DB (`scada_<slug>`) хранит данные проекта (события, тренды, пользователи).
- `projectSlug` извлекается из URL `/p/:slug/*` и прокидывается в контекст.

Ключевые элементы:
- `project/connection-manager.service.ts` — пул коннектов к проектным БД.
- `project/project.context.ts` — хранит slug и DSN в запросе.
- `scada-tcp/uid-router.service.ts` — по UUID станции ищет UID и связанные проекты.

Преимущества:
- Изоляция данных по проектам.
- Централизация конфигурации UID, повторное использование настроек.
- Горизонтальное масштабирование TCP/Web/API независимо.

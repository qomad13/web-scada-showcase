## SQL (Showcase)

Цель — показать модель данных без раскрытия реальных данных.

### Что положить сюда
- `schema_only/` — чистые схемы `CREATE TABLE` для каталога и проектов.
- `seed_demo.sql` — синтетический seed (demo UID, demo station, минимальные тренды/события).

### Что не класть
- Реальные дампы/данные/пароли.
- Исторические миграции с чувствительными именами/значениями.

### Пример структуры
```
sql/
├── schema_only/
│   ├── catalog_schema.sql      # только CREATE TABLE
│   └── project_schema.sql      # только CREATE TABLE
└── seed_demo.sql               # синтетический seed (заполнить по необходимости)
```

### Использование в демо
1) Импортировать схемы из `schema_only/`.
2) Накатить `seed_demo.sql` (синтетика).
3) Поднять backend с `.env` из примера (`../backend/ENV_EXAMPLE.md`).
